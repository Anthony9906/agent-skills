# 点胶机改机软件BOM发包

## 软件BOM组成

**完整软件BOM = 改机规则物料 + 视觉BOM表**

| 组成部分 | 来源 | 说明 |
|---------|------|------|
| 改机规则物料 | Agent根据规则自动生成 | 轴、胶阀、继电器、网线等 |
| 视觉BOM表 | 工程师提供 | 相机、镜头、光源等视觉组件 |

---

## 工作流程

```
1. 工程师提供改机参数
        ↓
2. Agent生成改机规则物料清单
        ↓
3. 工程师提供视觉BOM表
        ↓
4. Agent合并两部分 → 完整软件BOM
        ↓
5. 输出Excel格式的BOM报表
```

---

## 输入参数清单

工程师需提供以下改机参数：

| 序号 | 参数 | 说明 |
|-----|------|------|
| 1 | 轴数变化 | 格式：如"3轴改5轴"（增加2轴） |
| 2 | 胶阀类型 | N2400 / 5020R（高凯）/ EFD-PICO / PBA-Vulcan / PH30 / SPEN500 / PJXT300 |
| 3 | 增加气缸 | 数量 + 类型（单向/双向）+ 回流板/盲板/IO模块需求 |
| 4 | 翻转模组 | 有 / 无 |
| 5 | 改机源 | 项目号 + 站别 |
| 6 | 改造后 | 项目号 + 站别 |
| 7 | 新5轴 | 是否为新5轴改机（仅限868项目） |
| 8 | 镭射 | 有 / 无 |
| 9 | 发卡器 | 有 / 无 |

**若工程师未提供气缸详细信息（数量、类型、回流板等），需主动询问确认。**

## 视觉BOM表要求

工程师提供的视觉BOM表需包含以下字段：

| 字段 | 说明 |
|------|------|
| 料号 | 物料料号 |
| 名称 | 物料名称 |
| 数量 | 发包数量 |
| 品牌 | 物料品牌 |
| 型号 | 具体规格型号 |

---

## 核心规则速查

| 序号 | 规则 | 说明 |
|-----|------|------|
| 1 | 轴数增加 | 双龙门 → 所有轴相关物料×2 |
| 2 | 新5轴 | 仅发动力线和编码器线，CN1和驱动器沿用 |
| 3 | 胶阀串口线 | 每个改机BOM固定2根（前后龙门各1） |
| 4 | 胶阀通讯线 | 仅5020R（高凯）需发包2根 |
| 5 | 继电器 | 默认自制继电器（底座+上盖+PCBA×2）；PBA-Vulcan改为芬德继电器×2 |
| 6 | 驱动器串联网线 | 固定13根（11×0.5m + 1×5m + 1×3m） |
| 7 | 气缸 | 按实际数量发包电磁阀 |
| 8 | 工控机 | 根据源项目号判断（H36/H37沿用，H28/H29/H26/H27发I9-14代） |
| 9 | 接线盒 | H26/H27项目 + 三轴/四轴改五轴 → 2个 |
| 10 | 镭射 | 发适配器 + 485通讯线 |
| 11 | 发卡器 | 有则发包 |
| 12 | 比例阀网线 | H332站别 → 3米网线×1 |

## 质量控制机制

Agent在生成BOM表时，应遵循以下质量控制流程：

### 不确定时的处理流程

```
1. 生成BOM时遇到不确定的物料
        ↓
2. 检索规则文档（references/bom-rules.md）
        ↓
3. 是否找到明确规则？
    ├─ 是 → 按规则执行
    └─ 否 → 询问工程师确认
        ↓
4. 工程师确认后 → 更新BOM
        ↓
5. 输出最终BOM报表
```

### Agent应主动确认的情况

| 情况 | 处理方式 |
|------|---------|
| 物料发包规则不明确 | 询问工程师该物料是否需要发包 |
| 物料数量计算有歧义 | 询问工程师确认数量 |
| 料号/品牌/型号不完整 | 询问工程师提供完整信息 |
| 规则之间存在冲突 | 询问工程师优先级判断 |
| 工程师提供的参数不完整 | 主动询问补充信息 |

### 询问格式示例

当不确定时，Agent应清晰询问：

```
我不确定以下物料是否应该发包：
- 物料名称：[XXX]
- 可能触发的规则：[XXX]
- 我的判断依据：[XXX]

请问该物料是否需要发包？如果需要，数量是多少？
```
